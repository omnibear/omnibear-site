title: "Omnibear version 2.3.0"
date: 2026-07-08T06:34:07Z
author: Anthony Ciccarello
---

This release improves the loading of post types to more closely match the spec. If you have issues with either post types or syndication targets, ensure those are available from your micropub server's config query (as opposed to individual queries). Additionally, there were several bugs fixed around login and config loading.

This is also the first 2.x version to be submitted to the [Chrome Web Store](https://chrome.google.com/webstore/detail/omnibear/nhpadboepichgblikclgflndfdhjcnpi) (pending approval)

- Syndication targets and post types are now fetched using the Micropub `?q=config` query in [aa16bd8](https://github.com/omnibear/omnibear/commit/aa16bd8) (fixes [#164](https://github.com/omnibear/omnibear/issues/164))
- Fixed stale syndication targets being sent after logging out and back in in [7a9819f](https://github.com/omnibear/omnibear/commit/7a9819f) (fixes [#84](https://github.com/omnibear/omnibear/issues/84))
- Post types are now also cleared on logout in [7a9819f](https://github.com/omnibear/omnibear/commit/7a9819f)
- Fixed syndication targets and post types being fetched even when the token request failed in [ddfd968](https://github.com/omnibear/omnibear/commit/ddfd968) (fixes [#81](https://github.com/omnibear/omnibear/issues/81))
- Fixed a crash on the auth page when a token error occurred in [9cce597](https://github.com/omnibear/omnibear/commit/9cce597) (fixes [#82](https://github.com/omnibear/omnibear/issues/82))
- Bump vite from 8.0.14 to 8.0.16 by [@dependabot[bot]](https://github.com/dependabot) in [#162](https://github.com/omnibear/omnibear/pull/162)
- Bump undici from 7.26.0 to 7.28.0 by [@dependabot[bot]](https://github.com/dependabot) in [#163](https://github.com/omnibear/omnibear/pull/163)
- Additional dependency bumps in [f630418](https://github.com/omnibear/omnibear/commit/f630418)
